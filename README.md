# FastPrint Agent 专业打印服务解决方案

**告别端口映射 / 内网穿透 / 打印任务丢失**

**HTTP同步 + MQTT异步双协议打印中间件，完美适配本地内网 + 云端云服务 + 物联网分布式全场景**

---

## 一、项目简介

FastPrintAgent 是一款轻量、高性能的企业级分布式打印中间件，基于成熟 FastReport 报表引擎开发。

内置 HTTP 同步 + MQTT 异步双通信协议，兼顾传统本地对接与物联网云端分布式场景，完美补齐传统HTTP打印存在的内网穿透依赖、动态IP适配差、离线断连丢任务、多设备集群管控困难等短板，一套中间件可同时覆盖局域网本地打印、云端远程打印、工控物联网设备打印全业务场景。

程序支持 Windows 系统服务常驻、桌面客户端双模式运行，彻底解决企业异构系统打印集成复杂、多网点打印分散、报表开发冗余繁琐、多打印机调度不便、云端远程打印连通性差等行业痛点。内置可视化无代码模板设计能力，支持单据在线打印、在线预览、PDF导出归档，是企业信息化、工业自动化、物联网边缘计算场景下通用的标准化打印服务。

## 二、HTTP 与 MQTT 协议核心对比（核心差异化优势）

传统HTTP接口在分布式门店、工厂工控、云端远程设备场景中存在天然缺陷，本项目原生集成MQTT协议，针对性解决各类网络与打印任务痛点，两者详细对比如下：

| 对比维度 | 传统 HTTP 打印（同步调用） | 本项目 MQTT 打印（异步推送） |
| --- | --- | --- |
| 网络环境要求 | 必须固定内网IP、需要端口映射、必须暴露端口，存在网络安全风险 | 无需固定IP、无需端口映射、无需暴露本机端口，内网设备联网更安全 |
| 连接模式 | 短连接，请求完成立即断开，网络波动极易丢失打印任务 | 长连接常驻，自动断线重连，支持离线消息缓存，打印任务永不丢失 |
| 使用场景 | 仅限本地局域网、同网段业务系统对接 | 跨网段、跨城市连锁门店、云端远程下发、工控机、物联网终端 |
| 运维难度 | 每台设备单独配置网络，多设备集群运维流程繁琐 | 统一MQTT Broker管理，可批量管控所有打印终端，运维成本极低 |
| 适用人群 | 本地单一系统、局域网内网快速对接需求 | 分布式项目、云端项目、工控物联网、全国多门店集群项目 |

## 三、核心优势

- **双协议兼容，无缝平滑切换**：同一套打印模板、同一套业务JSON数据，无需修改业务代码，即可自由切换HTTP、MQTT两种调用方式
- **业务解耦，聚焦核心开发**：将打印能力从业务系统中完全抽离，业务系统无需关注打印机硬件、报表模板、底层打印驱动细节，极简对接即可完成全流程打印
- **企业级统一打印中台**：支持多套异构系统统一接入，规避重复开发打印业务逻辑，实现全网打印机统一调度、统一运维、统一监控
- **全开发语言兼容**：支持 C++、C#、Java、Python、Delphi、PHP、Go 等主流开发语言，统一JSON格式入参，无需适配底层驱动
- **多打印机集中管控**：通过接口可按需指定目标打印机，适配分布式环境下多设备、多打印机一站式调度管理
- **零代码可视化模板设计**：内置FastReport可视化设计器，拖拽即可生成报表，原生支持条码、二维码、电子签章、金额大写自动汇总功能
- **双运行模式稳定可靠**：支持Windows系统服务7×24小时无人值守运行，搭配桌面可视化客户端，同时满足生产环境稳定性与日常调试便捷性需求
- **一站式内置调试工具箱**：集成JSON格式化、Base64图片互转、全量日志追溯、一键模板设计功能，大幅降低接口对接与问题排查成本

## 六、快速开始

### 6.1 安装部署步骤

1. 下载 FastPrint Agent 安装包，解压至电脑纯英文路径，避免中文目录导致程序乱码异常
2. 右键以管理员身份运行「安装服务.bat」脚本，自动完成Windows系统服务安装
3. 服务安装后自动启动，默认HTTP监听端口：9798，支持自定义端口修改
4. 部署完成后程序常驻系统右下角托盘，左键点击图标即可打开客户端主界面进行参数配置
5. 进入计算机服务列表，查看 ASuperService 服务状态为正在运行，即代表部署成功

### 6.2 HTTP 调用基础规范

- 请求方式：HTTP POST
- 请求头：Content-Type: application/json
- 基础服务地址：http://127.0.0.1:9798

#### 6.2.1 HTTP最简调用示例

```json
{
  "printTemp": "SalesOrder",
  "printer": "",
  "printCn": 1,
  "printData": {
    "title": "销售订单",
    "orderNo": "SO20240520001",
    "customer": "张三",
    "items": [
      {"product": "笔记本电脑", "quantity": 1, "price": 5999},
      {"product": "鼠标", "quantity": 2, "price": 99}
    ]
  }
}
```

### 6.3 MQTT 调用规范（新增，含3个专属参数）

通过MQTT主题下发完整JSON请求报文，新增三个专属业务参数，支持自定义操作类型、唯一任务编号、异步回执主题，实现打印任务全流程异步闭环回调。

#### 6.3.1 MQTT完整请求示例

```json
{
  "printTemp": "SalesOrder",
  "printer": "",
  "printCn": 1,
  "printApi": "show-report",
  "printTaskId": "1234567",
  "responseTopic": "print/receive/1234",
  "printData": {
    "title": "销售订单",
    "orderNo": "SO20240520001",
    "customer": "张三",
    "items": [
      {"product": "笔记本电脑", "quantity": 1, "price": 5999},
      {"product": "鼠标", "quantity": 2, "price": 99}
    ]
  }
}
```

#### 6.3.2 MQTT异步统一状态回执（自动推送至指定回执主题）

```json
{
  "state": "success",
  "code": 200,
  "printTaskId": "1234567"
}
```

### 6.4 全功能示例查看

程序支持嵌套明细、金额自动汇总、金额大写转换、Base64电子签章渲染、PDF导出等全业务场景。打开 FastPrint Agent 客户端主界面 → 帮助 → JSON 示例，即可查看完整可直接复用的调试案例。

## 七、API 接口文档

### 7.1 HTTP 接口列表

| 接口功能 | 请求地址 |
| --- | --- |
| 直接打印 | http://127.0.0.1:9798/api/print-report |
| 打印预览 | http://127.0.0.1:9798/api/show-report |
| PDF 导出 | http://127.0.0.1:9798/api/export-pdf |

### 7.2 公共请求参数（HTTP/MQTT通用）

| 参数名 | 类型 | 必填 | 参数说明 |
| --- | --- | --- | --- |
| printTemp | string | 是 | FastReport 模板文件名，存放于程序 PrintTemplate 目录，仅传文件名称，无需携带 `.frx` 后缀 |
| printer | string | 是 | 目标打印机全称；调用优先级：指定打印机 > 程序默认打印机 > 系统默认打印机 |
| printCn | int | 是 | 打印份数，默认值为1 |
| exportFileName | string | 否 | 仅PDF导出接口生效，填写本地文件完整保存路径；<br>1. 仅适配 export-pdf 接口；<br>2. 需要保证目标目录拥有文件写入权限；<br>3. 调用示例：`"D:\\PDF\\test.pdf"`；<br>4. 为避免PDF中文乱码、文字空白、字体缺失问题，模板设计时所有中文文本控件，必须选用微软雅黑、宋体、黑体等系统兼容中文字体，禁止使用无中文适配的特殊字体 |
| printData | object | 是 | 1. 业务打印核心数据集；<br>2. 字段后缀添加 `_base64image`，程序可自动识别并渲染Base64格式图片；<br>3. 支持中文键名，模板可视化设计时可读性更强；<br>4. printData节点下不支持二次嵌套对象（Object），可支持多层数组（Array）嵌套，具体规范可参考客户端内置JSON示例 |

### 7.3 MQTT 专属新增参数（MQTT版本v5.0）

| 参数名 | 类型 | 必填 | 参数说明 |
| --- | --- | --- | --- |
| printApi | string | 是 | 对应接口执行能力：1. print-report直接打印 2. show-report打印预览 3. export-pdf导出PDF |
| printTaskId | string | 是 | 业务端自定义唯一任务编号，用于匹配异步回执消息，实现打印任务溯源 |
| responseTopic | string | 是 | 打印任务执行完成后，程序推送执行结果的自定义MQTT回执主题 |

### 7.4 统一响应参数说明

#### 7.4.1 成功响应

```json
{
  "printTaskId": "1234567",
  "state": "success",
  "code": 200
}
```

#### 7.4.2 失败响应

```json
{
  "printTaskId": "1234567",
  "state": "error",
  "code": 500,
  "message": "缺少printer字段"
}
```

## 九、模板设计教程

1. 打开客户端主界面，切换至打印调试页面
2. 粘贴业务JSON请求数据，或直接读取历史请求日志回填参数
3. 点击模板设计按钮，程序自动生成空白报表模板并绑定对应业务数据集
4. 拖拽表格、条码、二维码、图片、文本等控件，完成报表页面布局排版
5. 将页面控件绑定printData内对应业务字段，调整字体、页边距、分页规则
6. 保存模板文件，文件自动存入程序PrintTemplate目录，接口直接填写模板名即可调用
